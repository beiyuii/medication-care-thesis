概念原型与交互规划（不产出视觉稿，合并版）

一、页面结构与导航规划
- 页面列表：
  - 登录页：用户名登录；选择角色（老年人/护工/子女）。
  - 老年人首页：当日用药卡片、下一次提醒时间、开启检测入口、异常提示区。
  - 护工/子女首页：被照护人列表、状态摘要、异常汇总、历史记录。
  - 用药计划页：新增/编辑药品（四类）、剂量、频次、时间窗、周期；计划启停。
  - 摄像头检测页：授权摄像头、实时视图、检测反馈（药品/动作/组合）、手动确认按钮、语音提示开关。
  - 历史与统计页：时间筛选（日/周/月）、事件列表（含缩略图）、统计概览。
  - 异常与告警页：超时未确认、检测失败、权限异常；页面内高亮提示。
  - 设置页：隐私与权限、浏览器兼容提示、账户管理。
- 导航结构：顶部导航（角色端）、历史与统计、设置；可选侧边导航（首页/计划/检测/异常）；面包屑用于计划详情与事件查看回退。

二、关键流程与状态说明（务必手动确认）
- 计划创建：
  1) 老年人在用药计划页录入药品（四类）、剂量、频次、时间窗与周期。
  2) 保存后生成提醒任务；计划可启停与调整。
- 提醒与检测：
  1) 到达时间窗：页面进行语音播报提醒。
  2) 开启摄像头检测：实时识别药品目标与吃药动作。
  3) 事件判定：同时检测到药品与吃药动作 → 标记“疑似已服药”；仅其一 → 引导提示继续检测。
  4) 务必手动确认：用户点击“确认已服药”后，记录为“已服药”。
- 异常与告警：
  1) 时间窗结束仍未确认 → 记为异常事件，异常页高亮显示。
  2) 摄像头权限失败 → 提示开启权限或更换浏览器设置。
- 状态定义：
  - 检测状态：无检测/检测到药品/检测到动作/两者同时/权限异常/模型初始化中。
  - 事件状态：疑似已服药/已确认/异常（超时未确认）。

三、适老化交互规则
- 字体与对比度：正文≥16px，标题≥20px；深色文字配浅色背景，高对比度。
- 操作路径：核心任务 3 步内完成（开启检测→识别→手动确认）。
- 反馈清晰：状态条颜色+图标+简短文字；语音播报与页面高亮同步。
- 误触防护：确认按钮需要二次确认或短延时；大按钮与足够间距。
- 文案简洁：提示语≤15 字；避免专业术语。
- 权限引导：摄像头权限失败提供一键跳转浏览器设置的说明链接。
