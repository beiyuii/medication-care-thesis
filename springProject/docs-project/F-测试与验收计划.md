测试与验收计划（合并版，简版）

一、测试范围
- 算法：药品目标检测与吃药动作判定。
- 业务：提醒→检测→手动确认→记录；异常标记与查询。
- 适老化：字号、对比度、语音播报、二次确认防误触。
- 性能与兼容：端侧推理延迟、浏览器权限与回退。
- 安全与隐私：不上传视频，仅图片日志与事件。

二、关键用例
- 到时提醒：语音播报是否触发；静音/切换标签页下的提示。
- 摄像头权限：首次授权、拒绝后重新授权、权限缺失的引导。
- 检测状态：仅药品/仅动作/两者同时/无检测的反馈与引导。
- 手动确认：疑似已服药→点击确认→事件记录与页面更新。
- 异常：时间窗结束未确认→异常页高亮；查询与统计。
- 日志上传：疑似/确认/异常截帧上传成功，URL 可用。

三、验收指标（默认）
- 检测：mAP@0.5 ≥ 0.90；召回率 ≥ 0.92。
- 动作：准确率 ≥ 0.90；漏/误报 ≤ 5%。
- 性能：端侧延迟 ≤ 100ms/帧；权限流程稳定。
- 业务：提醒及时率 ≥ 99%；记录完整率 ≥ 99%。

四、测试数据与环境
- 数据：含多光照/角度/遮挡的四类药品样本；含吃药动作短序列。
- 环境：PC 摄像头 720p/30fps；主流浏览器（Chrome/Edge/Firefox/Safari）。
- 后端：本机 SQLite，日志图片目录可写；启用 JWT。

五、执行方式
- 步骤：准备数据→开启前端→登录 elder→执行计划→采集事件→压测推理延迟。
- 记录：用例编号、步骤、期望与实际、截图/日志 URL、结论。
- 缺陷：分级（阻断/重要/一般），定位与重测流程。

六、风险与回退
- 场景差异导致误判：增加样本与规则阈值调优。
- 权限异常：提供设置引导与失败回退页面。
- 性能波动：降分辨率与模型懒加载；回退至 WebGL/WASM。
