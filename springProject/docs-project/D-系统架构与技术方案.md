系统架构与技术方案（合并版，简版）

一、架构概览
- 前端：Vue3 + Vite + TypeScript + Pinia；端侧推理 onnxruntime-web（WebGPU/WebGL/WASM 回退）+ MediaPipe。
- 后端（本机）：springboot + mybatis-plus + mysql + lombok  + jwt + redis。
- 数据流：浏览器摄像头 → 端侧推理与规则融合 → 产生事件（疑似/确认/异常） → 上传图片日志与元数据 → 后端入库 → 管理端查看。

二、前端方案
- 模块：认证、用药计划、检测、事件与历史、异常、设置（适老化）。
- 推理流水线：权限→帧预处理→药品检测→关键点检测→动作判定→状态更新→截帧上传。
- 适老化：大字号、高对比度、语音播报、二次确认防误触。

三、后端方案（本机环境）
- 模块：Auth(用户名登录+JWT)、用户与角色、计划、事件、异常、日志图片、统计。
- 存储：mysql（结构化数据）；/logs 目录保存图片；记录文件名与 URL。
- 接口：REST 风格；鉴权中间件按角色限制读/写（护工/子女只读）。
- 运维：日志与错误栈；模型版本号由前端随事件上报便于追踪。

四、安全与隐私
- 视频仅前端处理，不上传；后端仅接收结构化事件与图片日志。
- HTTPS 已具备；JWT 有效期与刷新策略；最小权限访问。
- 用户授权可随时关闭摄像头；留存周期遵循常规网络文案。

五、部署与运行（示意）
- 前端：npm run build 输出静态资源；部署到本机 Web 服务或前端开发服务器。
- 后端：springboot 打包 jar 包；配置上传目录与 mysql 连接信息；环境变量管理密钥。
- 预置账号：用户名登录；初始化角色与关联关系。
