数据与模型方案（合并版）

一、识别对象与数据字典
- 目标类别：
  - PILL（药片/胶囊）：单粒或少量聚集的固体药品。
  - BLISTER（泡罩板）：铝塑板/泡罩包装，含多粒药片。
  - BOTTLE（药瓶）：塑料或玻璃瓶装药品。
  - BOX（药盒）：纸盒类外包装。
- 动作定义：
  - INTAKE_ACTION（吃药动作）：手-口接近并持续一定帧数，代表将药品送入口部的动作。
- 事件字段：event_id、patient_id、schedule_id、timestamp、status（suspected/confirmed/abnormal）、targets（类别与置信度）、action_detected（true/false）、image_log_url。
- 日志图片规范：上传关键帧图片（可含时间戳水印）；命名建议 patientId_scheduleId_timestamp.jpg。

二、数据采集与标注规范（PC 摄像头）
- 采集原则：分辨率≥720p、帧率≥25fps；多样光照/角度/遮挡；家庭环境与桌面材质多样化。
- 样本构成：静态图（各类药品摆放）、时序片段（吃药动作短视频或帧序列）。
- 标注规范：
  - 目标检测：边界框（x,y,w,h）+ 类别（PILL/BLISTER/BOTTLE/BOX）。
  - 动作标注：关键帧法（手接近口部的开始/结束帧）或规则辅助（连续帧满足距离阈值）。
- 数据划分：训练/验证/测试 ≈ 7:2:1，人员与场景尽量不交叉。
- 质量要求：误标率 ≤ 2%；类别均衡；遮挡与反光样本充足。

三、模型选型与事件判定规则
- 模型与库：
  - 药品检测：YOLOv8n/YOLOv10n 训练，导出 ONNX；前端用 onnxruntime-web（WebGPU 优先，回退 WebGL/WASM）。
  - 动作判定：MediaPipe Hands/Face/或轻量姿态 + 规则融合（手-口距离 + 持续帧数）。
- 默认阈值（可调）：
  - 目标检测：置信度 ≥ 0.35；NMS IoU ≤ 0.50；输入 640×640。
  - 动作判定：手-口距离阈值 d ≤ 8cm（像素近似）；连续满足帧数 ≥ 45 帧（≈1.5s@30fps）。
- 事件判定：
  - 时间窗内同时检测到药品与吃药动作 → 记“疑似已服药”；务必手动确认后才记“已服药”。
  - 仅其一成立 → 引导提示继续检测；时间窗结束未确认 → 记异常。
- 日志上传：疑似/已确认/异常事件均上传关键帧图片与元数据。

四、端侧推理集成方案
- 流水线：getUserMedia → Canvas 预处理 → ONNX 推理（药品）→ MediaPipe 关键点 → 规则融合 → 状态更新 → 截帧上传日志。
- 性能与兼容：WebGPU 优先，回退 WebGL/WASM；降分辨率与批次控制确保延迟 ≤100ms；资源释放与模型缓存。
- 安全与隐私：视频仅前端处理，不上传；后端仅接收图片日志与结构化事件。

五、模型评估指标与目标（默认）
- 检测：mAP@0.5 ≥ 0.90；召回率 ≥ 0.92；误报率 < 5%。
- 动作：准确率 ≥ 0.90；漏报 ≤ 5%；误报 ≤ 5%。
- 性能：端侧推理延迟 ≤ 100ms/帧；兼容主流浏览器权限与 API。
- 业务：提醒及时率 ≥ 99%；事件记录完整率 ≥ 99%；异常可追溯。
- 评测流程：独立测试集，覆盖多场景/多光照；对比不同浏览器性能与稳定性。
