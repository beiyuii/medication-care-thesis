## 前端整体风格设计思路

### 设计原则
- 强调适老化体验：核心任务三步可达、字体≥16px、语音与视觉同步反馈，避免专业术语。
- 保证检测链路可追溯：药品/动作/异常状态均以颜色+图标+文字三重反馈呈现。
- 兼顾安全与隐私：摄像头画面只在本地处理，日志上传有明确授权提示。

### 视觉语言
- **色板**：主色 `#2F6FED`（信任感）、强调色 `#FF9C41`（提醒）、状态色 `#4CAF50`（已确认）、`#FF5C5C`（异常）、`#F3B400`（疑似）、背景 `#F6F7FB`。
- **排版**：中文系统字体（思源黑体/苹方/微软雅黑），标题 24/20/18px，正文 16px，数据数字半粗体，行距 1.5。
- **栅格与间距**：12 栅格、1200px 内容宽度，卡片圆角 12px，内边距 24px，模块间距 16px，元素间距 8px，按钮高度 ≥52px。
- **图标与装饰**：线性图标 + 高对比底色，关键状态图标 24px，并在摄像头与提醒区域加脉冲动画提示。

### 组件与布局风格
- 顶栏使用浅色背景+阴影，左侧为系统名与角色切换，右侧展示语音与权限状态灯，次级标签用于历史/设置入口。
- 侧栏桌面端常驻并通过色块高亮当前页面，移动端抽屉式展开；悬停放大 1.05x 提升可感知性。
- 卡片组件左侧色条表示状态（绿确认、橙疑似、红异常、灰未开始），内部双列呈现药品信息与时间窗。
- 表单采用分组标题+步骤条，表格行高 ≥56px，尾列保留启停/编辑操作并拉开间距。
- 摄像头画布为深色背景+16px 圆角，覆盖状态层显示检测标签和帧率，右侧为提示语、语音开关与手动确认按钮（含 1 秒倒计时条）。
- 状态条固定在检测页顶部，颜色与文案同步并附“查看指引”链接。

### 页面级样式
- **登录页**：浅蓝到白的渐变背景，中置登录卡片，角色选择配图标说明，底部展示隐私与摄像头声明。
- **老年人首页**：大号数字展示下一次提醒，卡片化呈现今日计划/异常/检测入口，主按钮悬浮并实时显示权限状态。
- **护工/子女首页**：分栏表格显示关联老人的状态摘要、提醒进度与异常数量，支持快速筛选。
- **用药计划页**：卡片列表+抽屉表单，表单分“药品信息/时间窗口/频次周期”，附当天提醒时间轴可视化。
- **摄像头检测页**：左视频右面板布局，右侧展示检测结果 Tag、≤15 字提示语、语音开关、日志上传状态，底部“确认已服药”按钮带二次确认弹窗。
- **历史与统计**：顶部提供日/周/月筛选，中间时间线+图表（环图服药率、柱状漏服次数），点击事件打开抽屉查看 `targets_json` 与缩略图。
- **异常页**：大列表+状态标签+处理建议，顶部 banner 高亮超时未确认并提供跳转检测页操作。
- **设置页**：模块化卡片展示摄像头权限、语音播报、隐私说明、浏览器兼容提示，含操作按钮或链接。

### 交互与可访问性
- 关键按钮≥48px，键盘焦点清晰，摄像头组件提供 aria-label 描述当前状态。
- 文案保持 ≤15 字、语气口语化；确认操作附撤销或返回按钮。
- 语音播报与视觉提示同步，异常状态可附短暂闪烁但持续 ≤2 秒防疲劳。
- 权限失败时展示图文步骤与一键复制浏览器设置地址，并提供“查看教程”链接。

### 响应式与性能策略
- ≥1280px：侧栏常驻；768-1279px：侧栏缩为图标；<768px：栈式布局，检测页支持全屏模式。
- 摄像头组件可提示 WebGPU/WebGL/WASM 回退并在低性能模式降低分辨率，确保延迟目标 ≤100ms/帧。
- 借助 Tailwind token 对 spacing/typography/颜色统一管理，并通过 Naive UI 主题覆盖确保视觉一致。

### 验证指引
- 运行 `pnpm dev` 后按登录→首页→开启检测→疑似→确认→历史/异常链路穿行，核对状态色、倒计时、语音播报与日志提示。
- 在 Chrome/Edge/Firefox/Safari 复测摄像头权限、语音播报、响应式表现，并邀请目标用户进行可用性测试。
