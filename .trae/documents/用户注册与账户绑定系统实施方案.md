## 需求摘要
- 新增注册接口（用户名/密码/手机号/验证码校验，BCrypt 加密，返回用户ID与状态）
- 新增护工/子女账户绑定接口（绑定/解绑/查询，权限校验，维护关系表）
- 安全：HTTPS、敏感操作需认证、关键操作记录日志
- 性能：注册接口<500ms、支持1000+并发
- 测试：单元测试、压力测试、错误场景覆盖
- 文档：API文档、权限说明、错误码与异常处理

## 数据模型与库表
- users（现有，扩展字段）
  - 增加：`phone VARCHAR(20) UNIQUE`、`created_at DATETIME`、`updated_at DATETIME`
  - 索引：`UNIQUE(username)`、`UNIQUE(phone)`
- account_bindings（新表，用于护工/子女 ↔ 老人关系）
  - 字段：`id BIGINT PK AUTO_INCREMENT`、`elder_user_id BIGINT NOT NULL`、`related_user_id BIGINT NOT NULL`、`relation_type ENUM('caregiver','child') NOT NULL`、`status ENUM('bound','unbound') DEFAULT 'bound'`、`created_at DATETIME`、`updated_at DATETIME`
  - 索引：`INDEX(elder_user_id)`、`INDEX(related_user_id)`、`UNIQUE(elder_user_id, related_user_id, relation_type)`
- 验证码存储（推荐）：Redis（key：`sms:{phone}`，TTL：5min）或临时表 `sms_codes(phone, code, expires_at)`

## 接口设计
- 1) 账号注册
  - `POST /auth/register`
  - 请求体：`{ username, password, phone, code }`
  - 校验：
    - 非空与格式（用户名长度、密码强度、手机号正则、验证码长度）
    - 唯一性（username/phone）
    - 验证码核验（从 Redis/表读取，未命中或过期视为无效）
  - 处理：BCrypt 加密存储 `pwd_hash`，写入 `users`，返回 `{ userId, status: 'registered' }`
  - 响应：`ApiResponse<{ userId: number, status: string }>`
- 2) 护工/子女账户绑定
  - 绑定：`PUT /bindings`
    - 请求体：`{ elderUserId, relatedUserId, relationType, op }`（`op` 为 `bind|unbind`）
    - 认证：需登录，且操作者具备 `caregiver` 或 `child` 权限；或管理员
    - 处理：
      - `bind`：插入或更新 `account_bindings` 为 `bound`
      - `unbind`：更新 `status='unbound'`
  - 查询：`GET /bindings?elderUserId=&relationType=&page=&size=`
    - 返回：分页列表，包含绑定状态
- 3) 个人视角查询（可选增强）
  - `GET /bindings/me`：返回当前用户与老人绑定关系列表

## 校验与错误码
- 约定使用 `ErrorCode`：
  - `UNAUTHORIZED(401)` 未认证；`FORBIDDEN(403)` 无权限
  - `UNPROCESSABLE(422)` 参数错误；`NOT_FOUND(404)` 资源不存在
  - `SERVER_ERROR(500)` 服务器错误
- 表单校验失败返回 `422`，附加 `errors` 字段（字段→错误信息映射）

## 安全方案
- 强制 HTTPS：
  - 本地：建议通过反向代理（Nginx）或 Spring Boot SSL 自签证书（`server.ssl.*`）开启
  - 生产：配置真实证书与 HSTS
- 身份验证：
  - 使用现有 JWT；注册接口可开放（验证码必传以防滥用），绑定/解绑必须认证
- 审计日志：
  - 关键操作（注册、绑定/解绑）使用 `INFO` 级别记录操作者、目标ID、结果与耗时
  - 失败操作记录 `WARN/ERROR` 并附请求标识
- 保护策略：
  - 注册限流（IP/手机号级别，滑动窗口）
  - 验证码错误次数限制，黑名单短时封禁

## 性能方案
- 响应<500ms：
  - 索引保障唯一校验；Redis 校验验证码（O(1)）
  - 连接池与线程数适配（Hikari/Tomcat）
- 1000+并发注册：
  - 数据库主键与唯一索引，避免锁冲突；
  - 合理连接池大小（如 50–100），批量压测时控制 TPS，监测 p95
  - 日志异步/采样，避免阻塞

## 代码结构（类与方法）

  - `BindingRequest(elderUserId, relatedUserId, relationType, op)`

  - 扩展 `User` 增加 `phone` 字段；新增 `AccountBinding` 与 `AccountBindingMapper`
- 校验注解
  - DTO 上使用 `@NotBlank/@Pattern/@Size`
  - 统一异常处理返回错误详情（增强 `GlobalExceptionHandler`）
