# 数据库物理结构设计表

本文件根据当前项目 `schema.sql` 与实体代码整理，可直接作为论文第四章“数据库物理结构设计”部分的基础材料。

说明：

- 当前数据库为 MySQL 8。
- 现有脚本中大部分关联字段未显式声明外键约束，但业务层存在明确的逻辑关联关系，因此下表中的“FK”按逻辑外键口径描述。

## 可直接用于论文的小结

可在第四章数据库设计部分补充如下概述：

> 本系统数据库采用 MySQL 进行持久化存储。围绕“用户、患者、用药计划、提醒实例、服药事件、检测任务、异常告警”等核心对象进行表结构设计，以支持提醒驱动、检测回流、人工确认和异常处理等关键业务流程。其中，`schedules` 表用于保存计划模板，`reminder_instances` 表用于描述每日实际执行实例，`intake_events` 表用于记录检测和确认形成的服药结果，`alerts` 表用于保存超时未确认或检测失败等异常信息。
