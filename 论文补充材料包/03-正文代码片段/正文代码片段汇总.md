## 一、前端页面核心代码片段

建议放置位置：第五章“检测页面实现”小节。

作用说明：

- 前端在开启摄像头后截取当前画面；
- 将图像编码为 Base64；
- 调用检测接口并根据返回状态更新页面提示。

```ts
const runDetection = async (patientId: string, scheduleId: string) => {
  if (!isStreaming.value || isDetecting.value) return
  isDetecting.value = true

  const frame = await captureFrame()
  if (!frame) return

  try {
    const frameB64 = await frameToBase64(frame)
    const result = await predictDetection({
      patientId,
      scheduleId,
      timestamp: new Date().toISOString(),
      frameB64,
      cameraId: 'web-cam',
      modelVersion: 'web-yolo-v1',
    })
    detectionResult.value = {
      ...result,
      targetDetected: result.targets && result.targets.length > 0,
    }
  } finally {
    frame.close()
    isDetecting.value = false
  }
}
```

正文说明建议：

> 前端检测页面通过 `runDetection` 方法完成摄像头画面截取、图像编码和接口调用。系统在获取当前视频帧后，将其转换为 Base64 编码并提交给后端检测服务，再根据返回的检测状态和目标数量更新页面上的检测结果提示。

## 二、后端业务核心代码片段

建议放置位置：第五章“检测任务编排与告警处理实现”小节。

作用说明：

- Spring Boot 在收到检测结果后创建服药事件；
- 更新提醒实例状态；
- 当结果异常时自动生成告警。

```java
public void applyDetectionOutcome(
    ReminderInstance instance, Long detectionJobId, String resultStatus,
    Boolean actionDetected, String targetsJson, Timestamp occurredAt) {
  String normalizedStatus = normalizeInstanceStatus(resultStatus);
  IntakeEvent event = createEvent(instance, detectionJobId, normalizedStatus,
      Boolean.TRUE.equals(actionDetected) ? "hand_to_mouth" : "detected", targetsJson);
  event.setTs((occurredAt != null ? occurredAt : Timestamp.from(Instant.now())).toInstant().toString());
  intakeEventMapper.insert(event);

  instance.setStatus(normalizedStatus);
  instance.setLastEventId(event.getId());
  reminderInstanceMapper.updateById(instance);

  if ("abnormal".equals(normalizedStatus)) {
    createAlertIfMissing(instance, "detection_failed", "计划检测异常", "检测结果异常，请重新拍摄或人工确认。", "high");
  }
}
```

正文说明建议：

> 后端在接收到算法服务返回结果后，会先写入一条服药事件记录，再同步更新提醒实例的执行状态。当检测结果为异常时，系统还会自动生成对应的异常告警，从而形成“检测结果回写 - 状态更新 - 告警触发”的闭环处理机制。

## 三、算法调用核心代码片段

建议放置位置：第五章“目标检测与动作识别实现”小节。

作用说明：

- Flask 服务对视频逐帧结果进行聚合；
- 同时结合药品目标置信度与动作时间线；
- 最终输出 `confirmed`、`suspected`、`abnormal` 三态结果。

```python
def calculate_overall_status(self, targets, action_timeline):
    action_detected = len(action_timeline) > 0
    if not targets:
        return ("abnormal", 0.0, action_detected)

    avg_confidence = sum(t.score for t in targets) / len(targets)

    if action_detected and avg_confidence >= 0.85:
        status = "confirmed"
    elif action_detected or avg_confidence >= 0.5:
        status = "suspected"
    else:
        status = "abnormal"

    return (status, round(avg_confidence, 3), action_detected)
```

正文说明建议：

> 算法服务并不直接依据单帧结果给出最终判定，而是先对视频中的目标检测结果进行聚合，再结合动作识别得到的时间线信息计算综合状态。当药品目标和服药动作证据同时满足阈值条件时，系统判定为 `confirmed`；若仅满足部分条件，则判定为 `suspected`；否则判定为 `abnormal`。
