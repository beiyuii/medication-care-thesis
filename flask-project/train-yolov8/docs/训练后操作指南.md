# 训练完成后操作指南

## 一、训练完成后的文件位置

训练完成后，模型文件会保存在以下位置：

```
train-yolov8/outputs/runs/detect/medication_detection/
├── weights/
│   ├── best.pt          # 最佳模型（推荐使用）
│   ├── last.pt          # 最后一轮模型
│   └── best.onnx        # 导出后的ONNX模型（如果已导出）
├── results.png          # 训练曲线图
├── confusion_matrix.png # 混淆矩阵
└── ...
```

## 二、操作步骤

### 步骤1：查看训练结果

训练完成后，可以查看以下内容：

1. **训练曲线**：查看 `results.png` 了解训练过程
2. **混淆矩阵**：查看 `confusion_matrix.png` 了解各类别检测效果
3. **最佳模型指标**：在训练日志中查看 mAP、precision、recall 等指标

```bash
# 查看训练输出目录
ls -la outputs/runs/detect/medication_detection/
```

### 步骤2：导出ONNX模型

训练完成后，需要将 `.pt` 模型导出为 `.onnx` 格式，以便在生产环境中使用：

```bash
cd train-yolov8

# 使用默认路径导出（推荐）
python scripts/export_model.py

# 或指定模型路径
python scripts/export_model.py --model outputs/runs/detect/medication_detection/weights/best.pt

# 或指定输出路径
python scripts/export_model.py --output ../../models/medication-intake.onnx
```

**导出参数说明**：
- `--model`: 训练好的模型路径（默认：`outputs/runs/detect/medication_detection/weights/best.pt`）
- `--output`: 输出ONNX文件路径（默认：`../../models/medication-intake.onnx`）
- `--imgsz`: 输入图像尺寸（默认：640）

**导出成功后会显示**：
- ✅ ONNX模型已导出并复制到目标位置
- 📊 文件大小信息
- 💡 下一步操作提示

### 步骤3：验证模型文件

确认模型文件已正确导出到项目目录：

```bash
# 检查模型文件是否存在
ls -lh ../../models/medication-intake.onnx

# 查看文件大小（通常为几MB到几十MB）
```

### 步骤4：重启Flask服务

模型导出后，需要重启Flask服务以加载新模型：

```bash
# 如果服务正在运行，先停止（Ctrl+C）

# 重新启动服务
cd ../../flask-project
python -m app.main
```

**注意**：
- 服务启动时会自动加载 `models/medication-intake.onnx` 模型
- 首次加载可能需要几秒钟时间
- 可以通过健康检查接口确认模型是否已加载：`GET /health`

### 步骤5：测试模型效果

#### 5.1 使用测试页面

访问测试页面进行可视化测试：

```bash
# 启动服务后，在浏览器中访问
http://localhost:5000/test/video-detection
```

上传测试视频或图片，查看检测效果。

#### 5.2 使用API接口

**图片检测接口**：
```bash
curl -X POST http://localhost:5000/api/v1/detection/predict \
  -H "Content-Type: application/json" \
  -d '{
    "imageUrl": "https://example.com/test-image.jpg"
  }'
```

**视频检测接口**：
```bash
curl -X POST http://localhost:5000/api/v1/video-detection/detect \
  -F "video=@test-video.mp4"
```

## 三、模型性能评估

### 3.1 查看训练指标

训练完成后，在训练输出目录中查看：

- **results.png**: 包含 loss、mAP、precision、recall 等曲线
- **confusion_matrix.png**: 混淆矩阵，显示各类别的检测准确率
- **训练日志**: 在终端输出或日志文件中查看详细指标

### 3.2 评估建议

**良好的模型指标**：
- mAP@0.5 > 0.7（中等质量）
- mAP@0.5 > 0.8（良好质量）
- mAP@0.5 > 0.9（优秀质量）
- Precision > 0.8
- Recall > 0.8

**如果指标不理想，可以**：
1. 增加训练轮数：`--epochs 200`
2. 使用更大的模型：`--model s` 或 `--model m`
3. 增加数据量或提高数据质量
4. 调整数据增强参数

## 四、模型更新流程

如果需要更新模型：

1. **备份旧模型**（可选）：
   ```bash
   cp models/medication-intake.onnx models/medication-intake.onnx.backup
   ```

2. **导出新模型**：
   ```bash
   cd train-yolov8
   python scripts/export_model.py
   ```

3. **重启服务**：
   ```bash
   cd ../../flask-project
   python -m app.main
   ```

4. **验证新模型**：通过测试页面或API接口验证效果

## 五、常见问题

### Q1: 导出ONNX模型失败？

**可能原因**：
- 模型文件路径不正确
- ONNX依赖未安装

**解决方法**：
```bash
# 检查模型文件是否存在
ls -la outputs/runs/detect/medication_detection/weights/best.pt

# 安装ONNX依赖
pip install onnx onnxruntime

# 指定完整路径导出
python scripts/export_model.py --model <完整路径>/best.pt
```

### Q2: Flask服务无法加载模型？

**可能原因**：
- 模型文件路径配置错误
- 模型文件不存在
- ONNX Runtime未安装

**解决方法**：
```bash
# 检查模型文件是否存在
ls -lh models/medication-intake.onnx

# 检查配置文件
cat app/core/config.py | grep model_path

# 安装ONNX Runtime
pip install onnxruntime

# 检查服务日志
tail -f logs/app.log
```

### Q3: 检测效果不理想？

**可能原因**：
- 训练数据不足
- 模型未充分训练
- 数据质量有问题

**解决方法**：
1. 增加训练数据量
2. 增加训练轮数：`--epochs 200`
3. 使用更大的模型：`--model s` 或 `--model m`
4. 检查数据标注质量
5. 调整数据增强参数

### Q4: 如何比较不同训练结果？

**方法**：
1. 为每次训练指定不同的实验名称：
   ```bash
   python scripts/train.py --name medication_detection_v2
   ```

2. 比较不同实验的指标：
   ```bash
   # 查看不同实验的结果
   ls outputs/runs/detect/
   ```

3. 选择最佳模型进行导出
