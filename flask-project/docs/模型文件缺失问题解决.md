# 模型文件缺失问题解决方案

## 问题描述

错误信息：
```
Failed to load YOLOv8 model: [ONNXRuntimeError] : 3 : NO_SUCHFILE : 
Load model from models/medication-intake.onnx failed:
Load model models/medication-intake.onnx failed. File doesn't exist
```

## 问题原因

1. **模型文件未复制到正确位置**：训练完成后，ONNX模型文件在训练输出目录中，但没有复制到Flask项目目录
2. **路径解析问题**：代码中使用相对路径，但未正确解析为绝对路径

## 解决步骤

### 步骤1：复制模型文件

将训练好的ONNX模型文件复制到Flask项目目录：

```bash
# 方法1：手动复制（如果模型已导出）
cp train-yolov8/outputs/runs/detect/medication_detection/weights/best.onnx \
   flask-project/models/medication-intake.onnx

# 方法2：使用导出脚本（推荐）
cd train-yolov8
python scripts/export_model.py
```

### 步骤2：验证文件存在

```bash
ls -lh flask-project/models/medication-intake.onnx
```

应该看到类似输出：
```
-rw-r--r--  1 user  staff    12M Nov 18 20:07 medication-intake.onnx
```

### 步骤3：重启服务

```bash
cd flask-project
python -m app.main
```

## 代码修复

已修复 `app/services/video_detector.py` 中的路径解析问题：

**修复前**：
```python
session = ort.InferenceSession(
    self.settings.model_path,  # 相对路径，可能找不到文件
    providers=["CPUExecutionProvider"],
)
```

**修复后**：
```python
# 解析模型路径：如果是相对路径，转换为绝对路径
model_path = Path(self.settings.model_path)
if not model_path.is_absolute():
    # 获取项目根目录（app目录的父目录）
    app_dir = Path(__file__).parent.parent.parent
    model_path = app_dir / model_path

# 确保路径存在
if not model_path.exists():
    raise FileNotFoundError(f"模型文件不存在: {model_path}")

# 创建ONNX Runtime会话
session = ort.InferenceSession(
    str(model_path),  # 使用绝对路径
    providers=["CPUExecutionProvider"],
)
```

## 验证

修复后，服务启动时应该能够正常加载模型。可以通过以下方式验证：

1. **查看日志**：服务启动时应该看到：
   ```
   {"event": "yolo_model_loaded", "input_shape": [640, 640], ...}
   ```

2. **健康检查**：
   ```bash
   curl http://localhost:5000/health
   ```

3. **测试检测接口**：
   ```bash
   curl -X POST http://localhost:5000/api/v1/video-detection/detect \
     -F "video=@test-video.mp4"
   ```

## 预防措施

### 1. 训练完成后自动复制模型

可以在训练脚本中添加自动复制逻辑，或使用导出脚本。

### 2. 启动时检查模型文件

服务启动时检查模型文件是否存在，如果不存在给出明确提示。

### 3. 使用环境变量配置路径

可以通过环境变量 `ALGO_MODEL_PATH` 配置模型路径：

```bash
export ALGO_MODEL_PATH=/absolute/path/to/model.onnx
```

## 常见问题

### Q: 模型文件应该放在哪里？

A: 默认位置是 `flask-project/models/medication-intake.onnx`。可以通过配置文件或环境变量修改。

### Q: 如何更新模型？

A: 
1. 重新训练模型
2. 导出ONNX模型：`python scripts/export_model.py`
3. 重启Flask服务

### Q: 如何确认模型已加载？

A: 查看服务日志，应该看到 `yolo_model_loaded` 事件，而不是 `yolo_model_load_failed`。
