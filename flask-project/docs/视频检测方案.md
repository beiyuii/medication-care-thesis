# 视频检测方案设计文档

## 一、需求概述

### 1.1 业务需求
- **输入**：前端上传用户服药视频文件
- **输出**：检测结果（药品检测 + 服药动作识别）
- **移除**：不再支持单帧图片检测

### 1.2 技术目标
- 支持常见视频格式（MP4、AVI、MOV等）
- 视频预处理与帧提取
- 视频级检测结果聚合
- 保持低延迟和高准确率

## 二、模型推荐方案

### 2.1 方案对比

#### 方案A：YOLOv8 + 时序动作识别（推荐）
**优点**：
- YOLOv8 成熟稳定，社区支持好
- 实时性好，适合视频流处理
- 可导出 ONNX 格式，部署方便
- 支持多目标检测（PILL/BLISTER/BOTTLE/BOX）

**缺点**：
- 需要单独处理时序动作识别
- 需要帧级检测结果聚合

**适用场景**：需要高精度药品检测，动作识别通过时序规则判断

#### 方案B：VideoMAE + 动作分类
**优点**：
- 专门针对视频设计，能捕捉时序特征
- 端到端训练，动作识别效果好

**缺点**：
- 计算资源需求较高
- 需要大量视频数据训练
- 药品检测精度可能不如专用检测模型

**适用场景**：有充足训练数据，主要关注动作识别

#### 方案C：YOLOv8 + MediaPipe Hands（混合方案）
**优点**：
- YOLOv8 负责药品检测
- MediaPipe 负责手部关键点检测
- 通过手-口距离判断服药动作
- 轻量级，实时性好

**缺点**：
- 需要两套模型协同工作
- 手部检测可能受遮挡影响

**适用场景**：需要实时处理，对精度要求适中

### 2.2 采用方案：方案C（YOLOv8 + MediaPipe Hands）✅

**理由**：
1. YOLOv8 负责高精度药品检测（PILL/BLISTER/BOTTLE/BOX）
2. MediaPipe Hands 提供轻量级手部关键点检测
3. 通过手-口距离判断服药动作，准确且实时性好
4. 两套模型协同工作，各司其职，性能平衡
5. MediaPipe 无需额外训练，开箱即用

## 三、技术架构设计

### 3.1 视频处理流程（方案C）

```
前端上传视频
    ↓
后端接收（multipart/form-data）
    ↓
视频格式验证与转码
    ↓
视频帧提取（按FPS采样）
    ↓
并行处理：
  ├─→ YOLOv8：药品检测（PILL/BLISTER/BOTTLE/BOX）
  └─→ MediaPipe Hands：手部关键点检测
    ↓
动作识别：计算手-口距离，判断服药动作
    ↓
结果聚合与置信度计算
    ↓
返回检测结果
```

### 3.2 核心组件设计

#### 3.2.1 视频加载器（VideoLoader）
- 支持多种视频格式
- 视频元数据提取（时长、FPS、分辨率）
- 帧采样策略（均匀采样或关键帧提取）

#### 3.2.2 视频检测服务（VideoDetectorService）
- YOLOv8 模型加载与药品检测
- MediaPipe Hands 手部关键点检测
- 手-口距离计算与动作识别
- 批量帧检测与结果聚合

#### 3.2.3 检测结果聚合器（ResultAggregator）
- 帧级结果合并
- 置信度加权平均
- 时间窗口内动作判断

### 3.3 接口设计

#### 3.3.1 视频上传接口
```
POST /v1/detections/video/predict
Content-Type: multipart/form-data

请求参数：
- patientId: string (必需)
- scheduleId: string (必需)
- timestamp: datetime (必需)
- videoFile: File (必需，视频文件)
- cameraId: string (可选)
- modelVersion: string (可选)
- samplingRate: int (可选，默认30，每秒采样帧数)
- maxFrames: int (可选，最大处理帧数，默认300)
```

#### 3.3.2 响应结构
```json
{
  "status": "confirmed" | "suspected" | "abnormal",
  "confidence": 0.0-1.0,
  "actionDetected": boolean,
  "targets": [
    {
      "label": "PILL" | "BLISTER" | "BOTTLE" | "BOX",
      "score": 0.0-1.0,
      "bbox": [x, y, w, h],
      "firstDetectedFrame": int,
      "lastDetectedFrame": int,
      "detectionCount": int
    }
  ],
  "actionTimeline": [
    {
      "startFrame": int,
      "endFrame": int,
      "confidence": 0.0-1.0
    }
  ],
  "videoMetadata": {
    "duration": float,
    "fps": float,
    "totalFrames": int,
    "processedFrames": int
  },
  "latencyMs": int,
  "traceId": string
}
```

## 四、实现细节

### 4.1 视频处理库选择
- **OpenCV (cv2)**：视频读取、帧提取、格式转换
- **ffmpeg-python**：复杂格式处理（备选）

### 4.2 帧采样策略
- **均匀采样**：按固定间隔提取帧（如每0.5秒一帧）
- **关键帧提取**：基于场景变化检测（可选，计算成本高）

### 4.3 动作识别规则（方案C：MediaPipe Hands）

1. **手部关键点检测**：
   - 使用 MediaPipe Hands 检测手部21个关键点
   - 提取食指指尖（INDEX_FINGER_TIP）和拇指指尖（THUMB_TIP）位置
   - 计算手部中心点（手腕位置）

2. **口部位置估计**：
   - 使用 MediaPipe Face Mesh 或固定位置估算口部位置
   - 或使用手部位置与视频中心区域的关系推断

3. **手-口距离计算**：
   - 计算手部关键点（如食指指尖）到口部的欧氏距离
   - 距离阈值：≤ 8cm（像素近似，可配置）

4. **动作判定规则**：
   - 连续N帧（N ≥ 45，约1.5秒@30fps）满足：
     - 检测到药品（置信度 ≥ 0.35）
     - 手-口距离 ≤ 阈值
   - 满足条件则判定为"服药动作"

5. **时间窗口**：在指定时间窗口内（如3秒）同时满足药品和动作条件

### 4.4 性能优化
- **并发处理**：多帧并行检测
- **帧采样优化**：根据视频长度动态调整采样率
- **模型批处理**：批量推理提高吞吐量
- **缓存机制**：相同视频的检测结果缓存

## 八、风险评估与应对

### 8.1 风险点
1. **视频文件大小**：可能很大，影响上传和处理时间
   - 应对：前端压缩、限制文件大小、分块上传

2. **处理延迟**：视频处理比单帧慢
   - 应对：异步处理、进度反馈、采样优化

3. **存储成本**：视频文件存储需求大
   - 应对：处理完成后删除原始视频，只保留关键帧

4. **模型性能**：需要确保模型能处理视频帧
   - 应对：充分测试、性能监控、模型优化

### 8.2 兼容性考虑
- 保留图片检测接口一段时间，确保平滑迁移
- 提供配置开关控制新旧接口
