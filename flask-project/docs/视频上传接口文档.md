# 视频上传接口文档

## 接口概述

视频上传接口用于接收前端发送的视频文件，并将视频保存到服务器的 `video` 目录中。

## 接口列表

### 1. 上传视频文件

**接口地址**：`POST /v1/videos/upload`

**请求格式**：`multipart/form-data`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| videoFile | File | 是 | 视频文件 |
| patientId | String | 否 | 患者ID，用于生成文件名 |
| scheduleId | String | 否 | 计划ID，用于生成文件名 |
| customFilename | String | 否 | 自定义文件名（不含扩展名） |

**支持的视频格式**：
- `.mp4`
- `.avi`
- `.mov`
- `.mkv`
- `.flv`
- `.wmv`

**文件名生成规则**：
1. 如果提供了 `customFilename`，使用自定义文件名
2. 如果提供了 `patientId` 和 `scheduleId`，生成格式：`patient_{patientId}_schedule_{scheduleId}_{uuid}.mp4`
3. 否则，使用UUID生成唯一文件名：`{uuid}.mp4`

**响应格式**：

成功响应（200）：
```json
{
  "success": true,
  "filename": "patient_1_schedule_1_a1b2c3d4.mp4",
  "path": "video/patient_1_schedule_1_a1b2c3d4.mp4",
  "fullPath": "/path/to/flask-project/video/patient_1_schedule_1_a1b2c3d4.mp4",
  "size": 1024000,
  "traceId": "xxx-xxx-xxx"
}
```

错误响应（400/500）：
```json
{
  "error": "错误类型",
  "detail": "错误详情",
  "traceId": "xxx-xxx-xxx"
}
```

**示例请求**：

使用 curl：
```bash
curl -X POST http://localhost:5000/v1/videos/upload \
  -F "videoFile=@test-video.mp4" \
  -F "patientId=1" \
  -F "scheduleId=1"
```

使用 JavaScript (Fetch API)：
```javascript
const formData = new FormData();
formData.append('videoFile', videoFile);
formData.append('patientId', '1');
formData.append('scheduleId', '1');

fetch('http://localhost:5000/v1/videos/upload', {
  method: 'POST',
  body: formData
})
.then(response => response.json())
.then(data => {
  console.log('上传成功:', data);
})
.catch(error => {
  console.error('上传失败:', error);
});
```

使用 axios：
```javascript
import axios from 'axios';

const formData = new FormData();
formData.append('videoFile', videoFile);
formData.append('patientId', '1');
formData.append('scheduleId', '1');

axios.post('http://localhost:5000/v1/videos/upload', formData, {
  headers: {
    'Content-Type': 'multipart/form-data'
  }
})
.then(response => {
  console.log('上传成功:', response.data);
})
.catch(error => {
  console.error('上传失败:', error);
});
```

### 2. 列出已上传的视频

**接口地址**：`GET /v1/videos/list`

**查询参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| patientId | String | 否 | 过滤特定患者的视频 |
| scheduleId | String | 否 | 过滤特定计划的视频 |
| limit | Integer | 否 | 返回数量限制，默认100 |

**响应格式**：

成功响应（200）：
```json
{
  "success": true,
  "videos": [
    {
      "filename": "patient_1_schedule_1_a1b2c3d4.mp4",
      "path": "video/patient_1_schedule_1_a1b2c3d4.mp4",
      "fullPath": "/path/to/flask-project/video/patient_1_schedule_1_a1b2c3d4.mp4",
      "size": 1024000,
      "createdAt": 1703123456.789,
      "modifiedAt": 1703123456.789
    }
  ],
  "count": 1,
  "traceId": "xxx-xxx-xxx"
}
```

**示例请求**：

```bash
# 列出所有视频
curl http://localhost:5000/v1/videos/list

# 列出特定患者的视频
curl "http://localhost:5000/v1/videos/list?patientId=1"

# 列出特定计划的视频
curl "http://localhost:5000/v1/videos/list?scheduleId=1&limit=10"
```

## 配置说明

视频存储路径可以通过环境变量配置：

```bash
# 设置视频存储路径（相对于项目根目录）
export ALGO_VIDEO_STORAGE_PATH=video
```

默认路径：`video`（相对于项目根目录）

## 错误处理

### 常见错误

1. **文件格式不支持**
   - 错误码：400
   - 错误信息：`Unsupported video format. Supported formats: .mp4, .avi, .mov, ...`

2. **文件为空**
   - 错误码：400
   - 错误信息：`Video file is empty`

3. **缺少必需参数**
   - 错误码：400
   - 错误信息：`videoFile is required in multipart/form-data`

4. **文件保存失败**
   - 错误码：500
   - 错误信息：`Failed to save video file: ...`

## 使用场景

### 场景1：上传视频并保存

前端上传视频文件，后端保存到 `video` 目录：

```javascript
// 前端代码
const uploadVideo = async (file, patientId, scheduleId) => {
  const formData = new FormData();
  formData.append('videoFile', file);
  formData.append('patientId', patientId);
  formData.append('scheduleId', scheduleId);

  const response = await fetch('/v1/videos/upload', {
    method: 'POST',
    body: formData
  });

  const result = await response.json();
  if (result.success) {
    console.log('视频已保存:', result.path);
    return result;
  } else {
    throw new Error(result.detail);
  }
};
```

### 场景2：上传后立即进行检测

可以先上传视频，然后使用返回的文件路径进行检测：

```javascript
// 1. 上传视频
const uploadResult = await uploadVideo(file, patientId, scheduleId);

// 2. 使用保存的视频进行检测（如果需要）
// 注意：检测接口需要视频字节数据，可能需要先读取文件
```

### 场景3：列出患者的视频

```javascript
const listPatientVideos = async (patientId) => {
  const response = await fetch(`/v1/videos/list?patientId=${patientId}`);
  const result = await response.json();
  return result.videos;
};
```

## 注意事项

1. **文件大小限制**：当前没有设置文件大小限制，建议在生产环境中添加
2. **文件覆盖**：如果文件名已存在，会自动添加序号避免覆盖
3. **存储空间**：需要定期清理旧视频文件，避免占用过多存储空间
4. **安全性**：建议添加文件大小限制、文件类型验证、用户权限验证等安全措施
