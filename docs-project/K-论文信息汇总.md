## 2. 论文统一口径

- 模型推理位置：当前系统采用 Flask 服务端推理，不是浏览器端直接推理。
- 数据库：当前系统使用 MySQL，不是 SQLite。

- 统计逻辑边界：当前按“启用计划每天一个时间窗”进行约束化统计。

## 3. 数据集与实验信息

### 3.1 已确认内容
- 当前仓库保留的训练数据对应公开数据集 `Ultralytics medical-pills`。
- 当前可确认的数据集划分为：
  - 训练集：92 张图片
  - 验证集：23 张图片
  - 测试集：未单独划分或未保留
- 当前保留训练数据为公开单类药丸数据集，系统设计目标为四类药品对象。
- 标注方式：直接使用公开数据集原始标注。
- 采集设备写法：训练数据来自公开数据集，原始采集设备未在本项目仓库中单独记录；系统联调与演示阶段使用普通 PC 摄像头。
- 采集环境写法：公开数据集覆盖室内近景药丸拍摄场景；本系统联调与演示环境为普通室内桌面场景，使用 PC 摄像头，在自然光或常规室内灯光条件下进行测试。

### 3.2 论文建议写法
本研究前期采用 Ultralytics 提供的 medical-pills 公开数据集进行训练验证，其中训练集 92 张、验证集 23 张，未单独保留测试集。由于当前仓库中保留的数据主要为单类药丸检测样本，因此论文中应如实说明“当前保留训练数据为公开单类药丸数据集，系统设计目标为四类药品对象”，不宜将其表述为已经完成四类完整样本训练验证。

## 5. 算法实验结果

### 5.1 已确认的目标检测结果
- 论文采用“最佳轮次指标”作为目标检测实验结果。
- 最佳轮次：`epoch 86`
- Precision：`0.98719`，可写为 `98.72%`
- Recall：`0.98496`，可写为 `98.50%`
- mAP@0.5：`0.99348`，可写为 `99.35%`
- mAP@0.5:0.95：`0.77937`，可写为 `77.94%`

### 5.2 已确认的论文写法
- 动作判定模块当前采用基于手部关键点与规则融合的轻量方法，未单独形成标准准确率统计，主要在系统联调中验证其可用性。
- 误报率未单独形成标准实验统计，后续可在自建测试集上进一步评估。
- 漏报率未单独形成标准实验统计，后续可在自建测试集上进一步评估。
- 平均推理延迟暂按实验环境参考值描述，不作为标准算法结论。

## 6. 测试与验收材料

### 6.1 已可直接写入论文的结果
- 前端构建测试：`npm run build` 通过。
- Spring Boot 后端测试：`mvn test` 通过，共 7 个测试。
- Flask 算法服务测试：`pytest` 通过，共 7 个测试。
- 目标检测训练结果：最佳轮次 `epoch 86`，Precision `98.72%`，Recall `98.50%`，mAP@0.5 `99.35%`。

### 6.2 建议写入论文的手工验收场景
1. 用户登录系统后，能够按角色进入对应页面。
2. 老年用户能够创建并启用用药计划。
3. 检测页面能够完成一次视频录制并返回检测结果。
4. 检测结果能够生成服药事件并在历史记录中显示。
5. 超时未确认时系统能够显示异常告警，护工或子女端可以查看。

### 6.3 可直接使用的论文表述
系统测试采用自动化测试与手工场景验收相结合的方式进行。自动化测试主要覆盖前端构建、后端核心业务逻辑与算法服务接口；手工验收主要验证从登录、计划创建、检测识别、事件记录到告警展示的完整业务闭环。测试结果表明，系统在当前实验环境下具备较好的可运行性和业务完整性。
