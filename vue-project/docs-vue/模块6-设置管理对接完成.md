## ✅ 已对接接口

### 1. GET /api/settings - 获取设置

**功能描述**：返回当前登录用户的系统设置，如果不存在则返回默认值

**实现位置**：
- 服务层：`src/services/settingsService.ts` - `fetchUserSettings()`
- 视图层：`src/views/SettingsCenterView.vue` - `loadSettings()`

**数据映射**：
- 后端 `fallbackMode` (大写，如 "WASM") → 前端 (小写，如 "wasm")
- 其他字段直接映射

**权限说明**：
- 所有角色可读（需要鉴权）

---

### 2. PUT /api/settings - 更新设置

**功能描述**：更新当前用户的系统设置，仅老年人角色可写

**实现位置**：
- 服务层：`src/services/settingsService.ts` - `updateUserSettings(payload)`
- 视图层：`src/views/SettingsCenterView.vue` - `handleSubmit()`

**数据映射**：
- 前端 `fallbackMode` (小写) → 后端 (大写)
- 其他字段直接映射

**权限说明**：
- 仅 `elder` 角色可写
- 其他角色（`caregiver`、`child`）会返回 403 错误

**响应处理**：
- 成功后使用服务器返回的最新值更新表单
- 显示成功提示："设置已保存"

---

## 🔧 技术实现细节

### 数据映射函数

1. **`normalizeFallbackMode`**: 将后端返回的回退模式格式化为前端格式
   ```typescript
   'WASM' → 'wasm'
   'WEBGPU' → 'webgpu'
   'WEBGL' → 'webgl'
   ```

2. **`mapApiSettingToLocal`**: 将后端返回的设置数据映射为前端格式
   - 处理 `fallbackMode` 的大小写转换
   - 其他字段直接映射

3. **`mapLocalSettingToApi`**: 将前端的设置数据映射为后端请求格式
   - 将 `fallbackMode` 转换为大写
   - 其他字段直接映射

## 📝 注意事项

1. **权限限制**：
   - 只有 `elder` 角色可以更新设置
   - `caregiver` 和 `child` 角色只能查看设置
   - 如果非 `elder` 角色尝试保存设置，会收到 403 错误

2. **fallbackMode 格式**：
   - 后端使用大写格式（"WASM"、"WEBGPU"、"WEBGL"）
   - 前端使用小写格式（"wasm"、"webgpu"、"webgl"）
   - 数据映射函数会自动处理格式转换

3. **数据同步**：
   - 保存设置后，使用服务器返回的最新值更新表单
   - 确保前端显示的数据与服务器一致
