## 数据映射处理

### 后端 → 前端映射

1. **字段名映射**：
   - `dose` → `dosage`
   - `freq` → `frequency`
   - `winStart` → `window.start`
   - `winEnd` → `window.end`
   - `status: 'enabled'/'disabled'` → `status: 'active'/'paused'`

2. **ID类型**：后端返回 `number`，前端转换为 `string`（兼容现有代码）

3. **nextIntake字段**：后端未返回，前端暂时为空字符串（后续可能需要计算或从其他接口获取）

### 前端 → 后端映射

1. **字段名映射**：与后端→前端相反
2. **状态映射**：`'active'/'paused'` → `'enabled'/'disabled'`
3. **患者ID**：自动根据角色获取

## 患者ID获取逻辑

### elder角色
- 使用 `authStore.user.id`（自己的用户ID）

### caregiver/child角色
- 使用 `patientStore.activePatientId`（选中的患者ID）

### 错误处理
- 如果无法获取患者ID，抛出错误提示用户

## 统计数据获取

### 当前实现
- `fetchPlanStats()` 根据 `fetchSchedules()` 的结果计算统计数据
- 统计项：
  - 进行中：`status === 'active'` 的计划数量
  - 暂停中：`status === 'paused'` 的计划数量
  - 今日提醒：`status === 'active'` 的计划数量

### 后续优化
- 如果后端提供专门的统计接口，可以替换为直接调用接口
