# 错误处理优化说明

## 问题描述

后端返回的错误格式为：
```json
{
  "code": 422,
  "message": "用户名已存在",
  "data": null,
  "traceId": "c6273469-c127-4dcd-92d7-3c103018e5e2",
  "timestamp": "2025-11-15T09:10:58.507987Z"
}
```

但前端错误处理逻辑未能正确提取和展示 `message` 字段。

## 解决方案

### 1. 更新 HTTP 响应拦截器

**文件**：`src/lib/http.ts`

**改动**：
- ✅ 成功响应：检查 `code === 200` 才提取 `data`
- ✅ 错误响应：优先提取 `message` 字段（后端实际格式）
- ✅ 兼容多种错误格式：`message`、`msg`、`error`、`detail`
- ✅ 保留完整的错误信息：`status`、`code`、`traceId`、`timestamp`

```typescript
// 错误响应处理
const errorData = error.response?.data || {}
const errorMsg =
  errorData.message || // 优先使用 message（后端实际返回格式）
  errorData.msg ||    // 兼容 msg 字段
  errorData.error ||  // 兼容 error 字段
  errorData.detail || // 兼容 detail 字段
  error.message ||    // axios 错误消息
  '网络异常，请稍后重试'
```

### 2. 创建统一错误处理工具

**文件**：`src/utils/errorHandler.ts`（新建）

**功能**：
- `extractErrorMessage()`：统一提取错误消息
- `logError()`：开发环境下输出详细错误信息

**优势**：
- ✅ 统一的错误提取逻辑
- ✅ 支持多种错误格式
- ✅ 开发环境自动输出调试信息
- ✅ 类型安全

### 3. 更新所有视图文件

**更新的文件**：
- ✅ `src/views/LoginView.vue` - 登录/注册错误处理
- ✅ `src/views/PlanBoardView.vue` - 计划加载/操作错误处理
- ✅ `src/views/HistoryCenterView.vue` - 历史数据加载错误处理
- ✅ `src/views/SettingsCenterView.vue` - 设置加载/保存错误处理

**统一模式**：
```typescript
import { extractErrorMessage, logError } from '@/utils/errorHandler'

try {
  // ... 业务逻辑
} catch (error: unknown) {
  const errorMsg = extractErrorMessage(error, '默认错误消息')
  message.error(errorMsg)
  logError(error, '操作上下文')
}
```

## 错误消息提取优先级

1. **error.message** - HTTP 拦截器已处理的错误消息
2. **error.data.message** - 后端返回的 message 字段
3. **error.data.msg** - 兼容 msg 字段
4. **error.data.error** - 兼容 error 字段
5. **error.data.detail** - 兼容 detail 字段
6. **Error.message** - 标准 Error 对象
7. **默认消息** - 如果以上都不可用
