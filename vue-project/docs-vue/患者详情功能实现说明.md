# 患者详情功能实现说明

## 实现时间
2024-11-15

## 功能概述
在护工和子女首页添加"查看详情"功能，点击后通过 Modal 弹窗展示患者的完整信息，包括基础信息、用药计划（最近5条）和最近告警（最近5条）。

## 实现内容

### 1. 接口对接
- **接口**：`GET /api/patients/{id}`
- **服务函数**：`getPatientDetail(id: number)` in `src/services/patientService.ts`
- **状态**：✅ 已对接并实现

### 2. 功能位置
- **护工首页**：`src/views/CaregiverDashboardView.vue`
- **子女首页**：`src/views/ChildDashboardView.vue`

### 3. 功能特性

#### 3.1 查看详情按钮
- 位置：患者信息卡片右侧
- 状态：未选择患者时禁用
- 样式：主按钮样式（primary）

#### 3.2 Modal 弹窗展示
- **基础信息**：
  - 姓名
  - 年龄
  - 联系电话
  
- **用药计划（最近5条）**：
  - 药品名称
  - 剂量和频率
  - 服药时间窗口（开始时间 - 结束时间）
  - 周期
  - 类型
  - 状态（启用/禁用）
  
- **最近告警（最近5条）**：
  - 告警标题
  - 发生时间（格式化显示）

#### 3.3 交互体验
- 加载状态：显示加载动画
- 错误处理：加载失败时显示错误提示并关闭弹窗
- 空状态：无数据时显示友好提示

### 4. 代码修改

#### 4.1 `src/views/CaregiverDashboardView.vue`
- 添加 `getPatientDetail` 导入
- 添加 `PatientDetail` 类型导入
- 添加 Modal 相关状态管理（`showDetailModal`, `patientDetail`, `isLoadingDetail`）
- 添加 `handleViewDetail` 函数处理查看详情逻辑
- 添加 `formatDateTime` 函数格式化日期时间
- 添加"查看详情"按钮
- 添加患者详情 Modal 组件

#### 4.2 `src/views/ChildDashboardView.vue`
- 与护工首页相同的修改

### 5. 数据格式

#### 5.1 请求参数
```typescript
id: number  // 患者ID
```

#### 5.2 响应数据
```typescript
{
  id: number                    // 患者ID
  name: string                  // 患者姓名
  age: number                   // 年龄
  phone: string                 // 联系电话
  schedules: Array<{            // 用药计划（最近5条）
    id: number
    patientId: number
    medicineName: string
    type: string
    dosage: string
    frequency: string
    window: {
      start: string
      end: string
    }
    period: string
    status: string              // 'enabled' | 'disabled'
  }>
  recentAlerts: Array<{         // 最近告警（最近5条）
    id: number
    title: string
    occurredAt: string         // ISO8601 格式
  }>
}
```

### 6. 错误处理
- 使用 `extractErrorMessage` 统一提取错误消息
- 使用 `logError` 记录详细错误信息（开发环境）
- 加载失败时自动关闭 Modal

## 使用说明

### 护工角色
1. 登录后进入护工首页
2. 选择要查看的患者（下拉选择框）
3. 点击"查看详情"按钮
4. 在弹窗中查看患者完整信息

### 子女角色
1. 登录后进入子女首页
2. 选择要查看的家人（下拉选择框）
3. 点击"查看详情"按钮
4. 在弹窗中查看家人完整信息

## 技术细节

### 类型安全
- 使用 TypeScript 类型定义确保类型安全
- `PatientDetail` 接口与后端响应格式一致

### 性能优化
- 仅在点击"查看详情"时加载数据
- Modal 关闭后不保留数据，下次打开重新加载

### 用户体验
- 加载状态明确
- 错误提示友好
- 空状态提示清晰
- 时间格式化符合中文习惯
