## ✅ 已对接接口

### 1. GET /api/alerts - 查询告警列表

**功能描述**：根据患者ID查询该患者的所有异常告警记录

**实现位置**：
- 服务层：`src/services/alertService.ts` - `fetchAlerts(patientId?)`
- 视图层：`src/views/AlertCenterView.vue` - `loadAlerts()`

**数据映射**：
- 后端 `severity` (`high`/`medium`/`low`) → 前端 (`danger`/`warning`/`info`)
- 后端 `status` (`pending`/`resolved`) → 前端 `resolved` (boolean)
- 后端 `ts` (ISO8601) → 前端 `occurredAt` (格式化后的中文字符串)
- 后端 `actionNote` → 前端 `suggestion`（如果为空则根据 `type` 生成默认建议）

**患者ID获取逻辑**：
- `elder` 角色：使用自己的 `userId`
- `caregiver`/`child` 角色：使用 `patientStore.activePatientId`
- 支持手动传入 `patientId` 参数

---

### 2. POST /api/alerts/{id}/resolve - 标记告警已处理

**功能描述**：将告警状态更新为 `resolved`，并记录处理时间和处理备注

**实现位置**：
- 服务层：`src/services/alertService.ts` - `resolveAlert(alertId, actionNote?)`
- 视图层：`src/views/AlertCenterView.vue` - `handleResolve(alert)`

**请求参数**：
- `alertId` (string): 告警ID（路径参数）
- `actionNote` (string, 可选): 处理备注

**响应处理**：
- 成功后显示成功提示
- 自动刷新告警列表以获取最新状态

---

## 🔧 技术实现细节

### 数据映射函数

1. **`mapSeverity`**: 将后端严重程度映射为前端格式
   ```typescript
   'high' → 'danger'
   'medium' → 'warning'
   'low' → 'info'
   ```

2. **`formatTimestamp`**: 格式化时间戳为可读格式
   ```typescript
   '2024-11-13T20:30:00Z' → '2024/11/13 20:30'
   ```

3. **`generateSuggestion`**: 根据告警类型生成处理建议
   ```typescript
   'timeout' → '请致电确认是否已服药'
   'detection_failed' → '请检查摄像头和网络连接'
   'missed' → '请关注患者是否漏服，必要时联系家属'
   ```

4. **`getPatientName`**: 获取患者名称
   - `elder` 角色：从 `authStore.user.name` 获取
   - `caregiver`/`child` 角色：从 `patientStore.activePatient.name` 获取

## 📝 注意事项

1. **患者名称获取**：
   - 在服务层映射时，`getPatientName()` 依赖于 `patientStore` 已加载患者列表
   - 如果患者列表未加载，可能显示"未知患者"
   - 建议在视图层确保患者列表已加载后再调用 `fetchAlerts`

2. **处理建议生成**：
   - 如果后端返回的 `actionNote` 为空，前端会根据 `type` 生成默认建议
   - 默认建议映射表在 `generateSuggestion` 函数中定义
   - 如需添加新的告警类型建议，请更新该函数

3. **时间格式**：
   - 后端返回的时间为 ISO8601 格式（如：`2024-11-13T20:30:00Z`）
   - 前端显示格式为中文本地化格式（如：`2024/11/13 20:30`）
   - 格式化逻辑在 `formatTimestamp` 函数中
