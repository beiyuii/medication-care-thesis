### 2. 新增认证类型定义

**文件**：`src/types/auth.ts`（新建）

**内容**：
- `LoginRequest`：登录请求参数
- `RegisterRequest`：注册请求参数
- `ApiResponse<T>`：后端统一响应格式
- `LoginResponse`：登录响应数据
- `UserProfileResponse`：用户信息响应数据
- `PatientSummary`：患者摘要信息

**用途**：为认证相关接口提供类型安全

### 3. 更新 HTTP 客户端

**文件**：`src/lib/http.ts`

**改动**：
1. **响应拦截器**：适配后端统一响应格式 `{ code, msg, data, traceId, timestamp }`
2. **错误处理**：提取 `error`、`detail`、`traceId` 字段
3. **401 处理**：自动跳转登录页并清除会话

```typescript
// 响应拦截器（核心改动）
http.interceptors.response.use(
  (response: AxiosResponse) => {
    // 提取 data 字段返回
    return response.data?.data ?? response.data
  },
  error => {
    // 401 时清除会话并跳转登录
    if (error.response?.status === 401) {
      authStore.clearSession()
      window.location.href = '/login'
    }
    // 统一错误格式
    return Promise.reject({
      status: error.response?.status ?? 0,
      message: errorMsg,
      traceId: error.response?.data?.traceId,
    })
  }
)
```

### 4. 新增认证服务

**文件**：`src/services/authService.ts`（新建）

**接口对接**：
- `login(payload)`：`POST /auth/login`
- `register(payload)`：`POST /auth/register`
- `getProfile()`：`GET /auth/profile`

**示例代码**：
```typescript
export async function login(payload: LoginRequest): Promise<LoginResponse> {
  return http.post('/auth/login', payload)
}

export async function register(payload: RegisterRequest): Promise<LoginResponse> {
  return http.post('/auth/register', payload)
}
```

### 5. 实现 Token 持久化

**文件**：`src/stores/auth.ts`

**改动**：
1. **初始化 state**：从 localStorage 读取认证信息
2. **setSession**：保存 token、user、role 到 localStorage
3. **clearSession**：清除 localStorage 中的认证信息

**持久化字段**：
- `auth_token`：JWT token
- `auth_user`：用户信息 JSON
- `auth_role`：角色字符串

**效果**：刷新页面后仍保持登录状态

### 6. 新增注册功能

**文件**：`src/views/LoginView.vue`（重构）

**新增功能**：
1. **表单模式切换**：登录 ⇄ 注册
2. **注册表单**：
   - 用户名
   - 密码
   - 角色选择（elder/caregiver/child）
3. **登录表单**：
   - 用户名
   - 密码
   - 记住登录状态
4. **错误提示**：使用 Naive UI 的 message 组件
5. **表单验证**：用户名不为空、密码不少于6位

**交互流程**：
```
登录页 
  ↓ 点击"立即注册"
注册表单（含角色选择）
  ↓ 填写并提交
后端验证 → 返回 token
  ↓
保存到 store + localStorage
  ↓
跳转到对应角色首页
```

### 7. 路由配置更新

**文件**：`src/router/index.ts`

**改动**：
- 将 `/family/home` 改为 `/child/home`
- 更新路由元信息中的角色类型
- 更新 `allowRoles` 数组

### 8. 视图文件重命名

**改动**：`FamilyDashboardView.vue` → `ChildDashboardView.vue`

**原因**：与角色类型统一保持一致
