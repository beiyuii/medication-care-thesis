# 用户个人信息查看功能说明

## 功能概述

用户可以通过两种方式查看个人信息：
1. **设置页面**：在设置页面顶部显示"账户信息"卡片
2. **顶部导航栏**：点击右上角"账户"下拉菜单中的"个人信息"选项

## 功能特性

### 显示的信息

1. **基本信息**：
   - 用户ID
   - 用户名
   - 姓名
   - 角色（老年人/护工/子女）

2. **关联患者信息**（仅护工/子女角色显示）：
   - 患者姓名
   - 下次服药时间
   - 计划状态（进行中/已暂停）
   - 告警数量

## 使用方法

### 方法一：通过设置页面查看

1. 登录系统后，点击左侧导航栏的"设置"
2. 在设置页面顶部可以看到"账户信息"卡片
3. 卡片中显示完整的用户信息和关联患者列表

### 方法二：通过顶部导航栏查看

1. 登录系统后，点击右上角的"账户"按钮
2. 在下拉菜单中选择"个人信息"
3. 自动跳转到设置页面，并显示账户信息

### 角色显示说明

- **老年人角色**：显示"当前账户为老年人角色，无需关联患者"
- **护工/子女角色**：显示关联的患者列表，包括：
  - 患者姓名
  - 下次服药时间（格式化显示）
  - 计划状态标签（绿色=进行中，黄色=已暂停）
  - 告警数量（如果有告警，显示红色提示）

## 技术实现

### 接口调用

- **接口**：`GET /api/auth/profile`
- **服务函数**：`getProfile()` from `src/services/authService.ts`
- **响应类型**：`UserProfileResponse`

### 数据流程

```
用户访问设置页面
  ↓
调用 getProfile() 接口
  ↓
后端返回用户信息（包含关联患者）
  ↓
前端展示在"账户信息"卡片中
```

### 错误处理

- 如果加载失败，显示错误提示
- 开发环境下输出详细错误信息到控制台
- 使用统一的错误处理工具 `extractErrorMessage`
